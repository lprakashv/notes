# SQLite

## In-a-nutshell

- An in-process library -> not a server
- 1 file of ANSI-C code
- Database is a single file
  - can contains 1000s of tables, indexes and views
  - space efficient
- Full-featured SQL
- Power-safe serializable transactions
- Fast
- Simple API
- Public domain

## "Lite" means "low overhead", but not "low capability"

- 1 writer + N concurrent readers
- 1 GB strings and BLOBs
- 281 TB databases
- 64-way joins
- 2000 columns per table or index
- No arbitrary limits on numbers of:
  - tables
  - indexes
  - rows in a table

## Not for

- Remote data
- Big data
- Concurrent writers

```text

| SQL | -->  | compiled SQL into program | --> | prepared statement | --> | run the program | --> | result |
+-----+      +---------------------------+     +--------------------|     +-----------------+     +--------+

             \___________  ______________/                                \_______  _______/
                         \/                                                       \/

```

Parser

- LALR(1) parser generated by "Lemon"
- Reentrant and threadsafe
- Output: is an AST
- Hand-written tokenizer

Code generator

- Semantic analysis
- AST transformations
- Query planning
- Output: prepared statement (bytecode)

Virtual machine

- bytecode interpreter
- 3-address, 5-operand register machine
- Bit `switch` statement on the `opcode` and inside a loop
- Special opcodes designed to help implement SQL statements

Storage Engine Summary

- Row store
- Variable length entries
- Forest of B-trees
  - 1 b-tree for each table and each index
  - table key: PRIMARY KEY or ROWID
  - index key: indexed columns + table key
- Transaction control using "rollback-journal" or "write-ahead-log" (WAL)

B-Tree

- B Trees and B+ trees
- multiple B trees per disk file
- variable length entries
- free page tracking and reuse
- access via cursor
- concurrent read/write of the same table using separate cursors

Pager

- atomic commit or rollback
  - power safe transactions
  - rollback mode or WAL mode
- uniform size pages numbed from 1
- 512 or 65536 bytes per page
- no interpretation of page content
- cache
- concurrency control

OS Interface (`os_unix.c` / `os_win.c`)

- platform specific interface to the OS
- runtime changeable
- portability layer
- `read()`/`write()` or `mmap()`
- `https://sqlite.org/vfs.html`
- direct i/o to hardware: `test_onefile.c`

VFS Shims

- inserted in-between pager and os interface
- encryption
- compression
- logging
- testing and fault injection
- ...
